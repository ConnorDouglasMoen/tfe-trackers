SYSTEM: Tales from Elsewhere (TFE)
--------------------------------------------------------------------

HP SYSTEM: Strain + Injuries
- Strain: integer 1-9. Survivors have strain 3 by default (max 9).
  Represented as checkboxes (filled = taken, shown as dark red X).
- Injuries: up to 3 types per character (Survivors have all 3):
    * Serious Injuries: up to 2 slots
    * Critical Injury:  1 slot
    * Lethal Injury:    1 slot
  Each injury slot has:
    - Location (text + CLEAR button)
    - Complications (text + CLEAR button; faded + strikethrough when Treated)
    - Treated toggle

CONDITIONS:
- Temporary statuses. Stored as string[]. Enter to add, X to delete.

CHARACTER TYPES:
- Survivors (Player Characters): strain 3, all injury types
- Other (NPCs): minimal defaults, +/- controls to add injury tiers
- Switching type preserves each blob independently (no data bleed)

DATA MODEL
--------------------------------------------------------------------

--- Token metadata: TokenRecord ---

type CharacterType = "survivor" | "other";

type InjurySlot = {
  id: string;
  location: string;
  complications: string[];  // list, same pattern as conditions
  treated: boolean;
};

type CharacterData = {
  characterType: CharacterType;
  strainMax: number;         // 1-9
  strainCurrent: number;     // 0-strainMax
  hasSerious: boolean;
  seriousCount: 1 | 2;
  hasCritical: boolean;
  hasLethal: boolean;
  seriousInjuries: [InjurySlot, InjurySlot];
  criticalInjury: InjurySlot;
  lethalInjury: InjurySlot;
  conditions: string[];
};

type TokenRecord = {
  activeType: CharacterType;
  survivor: CharacterData;   // always preserved, even when "other" is active
  other: CharacterData;      // always preserved, even when "survivor" is active
};

Defaults:
  Survivor — strainMax:3, all injuries, seriousCount:2
  Other    — strainMax:1, hasSerious:true, seriousCount:1

Migration (migrateToTokenRecord):
  - Handles legacy CharacterData blobs (pre-TokenRecord)
  - Fills in characterType, seriousCount
  - Strips legacy displaySettings (now in scene metadata)
  - migrateInjurySlot: converts legacy single-string complications → string[]
  - migrateCharacterData: also normalises legacy string conditions → string[]

--- Scene metadata: DisplaySettings ---

type DisplaySettings = {
  showStrain: boolean;
  showConditions: boolean;  // also gates complication bubbles on map
  // "all" = empty + filled circles; "filled-only" = filled only; "none" = hide row
  injuryDisplay: "all" | "filled-only" | "none";
};

Stored at: OBR.scene.metadata[getPluginId("sceneDisplaySettings")]
Written by: GM only (Action panel)
Read by: all participants via OBR.scene.onMetadataChange
Defaults: showStrain:true, showConditions:true, injuryDisplay:"all"

--- Per-token: TokenDisplayOverrides ---

type TokenDisplayOverrides = {
  showStrain: boolean | null;          // null = use scene setting
  showConditions: boolean | null;      // null = use scene setting
  injuryDisplay: "all"|"filled-only"|"none"|null; // null = use scene setting
};

Stored inside TokenRecord.displayOverrides on each token.
Editable by all users (players + GM) in the token context menu.
Resolution: resolveDisplaySettings(sceneSettings, tokenOverrides)
  - Non-null token value wins; null falls back to scene setting.
Default: all fields null (fully inherits from scene).

--------------------------------------------------------------------
ON-MAP DISPLAY
--------------------------------------------------------------------
Implemented in src/background/:
  onMapDisplay.ts   — init; fetches DPI via OBR.scene.grid.getDpi();
                      reads DisplaySettings from scene metadata;
                      full redraw on metadata change
  onMapHelpers.ts   — buildStrainItems, buildInjuryItems (takes
                      DisplaySettings arg), buildConditionItems
  onMapItemIds.ts   — deterministic ID generators + getAllAttachmentIds
  mathHelpers.ts    — getImageCenter (copied from owl-trackers)

Layout:
  Strain + Injury rows: rendered BELOW token (getTokenBottomLeft)
  Bubble row:           rendered ABOVE token (getTokenTopLeft)
    - Bubbles wrap to fit token width, left-aligned.
    - Width measured via canvas.measureText for accurate sizing.

  Row 1 (below): Strain boxes (13px squares, red; dark red X when filled)
  Row 2 (below): Injury circles (14px, color-coded by severity)
                   ○ = empty slot (shown only when injuryDisplay = "all")
                   ! = filled + untreated
                   ✓ = filled + treated
  Row 3 (above): Text bubbles — untreated complications + conditions
                   Both gated by showConditions (turning it off hides all bubbles)
                   Untreated complications: color = injury severity
                   Conditions:              white (#ffffff)
                   Dedup: same string from multiple sources → shown once,
                     color of highest severity (lethal > critical > serious > condition)
                   Background: #1a1a2e at 0.8 opacity; pill-shaped via buildPath

Injury circle colors:
  Serious  — fill #d68e68, stroke #a44f27
  Critical — fill #db7777, stroke #94272c
  Lethal   — fill #7c4291, stroke #501e64

DPI: fetched via OBR.scene.grid.getDpi() on each scene ready; cached
as module-level `sceneDpi` and passed to all builder helpers.

onMapDisplay.ts trigger for bubble row:
  Shows bubbles when showConditions=true AND:
    (any untreated injury has complications) OR (data.conditions.length > 0)

--------------------------------------------------------------------
DECISIONS MADE
--------------------------------------------------------------------
- Survivor/Other are fully independent blobs in TokenRecord; switching
  type only changes activeType, both blobs always preserved.
- DisplaySettings live in OBR scene metadata, not on tokens. GM-only
  UI in token menu. Changes affect all tokens for all participants.
- Strain boxes use inline styles (not Tailwind) to avoid purge issues.
- Strain default for Survivor: 3.
- "Open Full Editor" hidden when isPopover=true (?popover=1 URL param).
- Conditions stored as string[]; legacy string migration handled.
- Complications stored as string[] (same pattern as conditions).
  Legacy single-string complications migrated to [string] on load.
- Separate embed heights for player (420px) and GM (560px).
- Token overrides (TokenDisplayOverrides) stored in TokenRecord.displayOverrides.
  null = inherit from scene; non-null = override for that token only.
  resolveDisplaySettings() merges scene + token overrides before building map items.
- Bubble text width measured via canvas.measureText (not estimated).
- Tracked Token IDs stored at: OBR.scene.metadata[getPluginId("trackedTokenIds")]
  - string[] of OBR item IDs pinned by the GM
  - Toggled via context menu "Pin to Action Panel" (background.ts onClick)
  - Unpin button in TrackedTokenRow calls useTrackedTokensStore.untrackToken
  - Action panel reads list, resolves live Items from useOwlbearStore.items
  - Deleted items are filtered out silently (no stale IDs shown)
- writeTokenRecordToItem(itemId, record) added to itemMetadataHelpers.ts;
  used by TrackedTokenRow so each row writes directly to its target item
  without needing or affecting the current selection.
- TrackedTokenRow re-syncs local record from item prop on each items change
  so concurrent edits (player context menu + GM Action panel) stay in sync.

- Bubbles render ABOVE token (getTokenTopLeft), not below.
